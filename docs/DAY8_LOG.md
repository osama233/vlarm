## 关键发现与修复

### 1. DDIM 数值爆炸（重大 Bug 修复）

**问题**: DDIM 推理输出范围 ±400（GT 范围 ±3），完全无法使用。

**根因**: Cosine β schedule 的 ᾱ_99 = 2.4×10⁻⁷ ≈ 0。DDIM 公式中有除以 √(ᾱ_t) 的操作：

```
pred_x0 = (x_t - √(1-ᾱ_t)·ε_θ) / √(ᾱ_t)
```

当 ᾱ_t ≈ 0 时，√(ᾱ_t) ≈ 0.0005，将噪声预测误差放大 **2000 倍**。

即使噪声预测 MSE=0.06（85% 优于随机），一步 DDIM 的系数 `50*x_t - 49*ε_θ` 需要 x_t 和 ε_θ 精确对齐才能抵消，否则输出范围被放大 50×。

**修复**: 改用 **DDPM 采样**（100 步，不除以 √(ᾱ_t)，数值稳定）

| 采样方法 | 步数 | 数值稳定性 | 输出范围 | 推理时间 |
|---------|------|-----------|---------|---------|
| DDIM (修复前) | 16 | ❌ 爆炸 | ±800 | 3.1s |
| DDIM (clip修复) | 16 | ❌ 仍爆炸 | ±300 | 3.4s |
| **DDPM** | **100** | **✅ 稳定** | **±3 (匹配GT)** | **16.9s** |

DDPM 系数：`1/√(α_t) · (x_t - β_t/√(1-ᾱ_t)·ε_θ)` — 其中 α_t = 1-β_t ≈ 1，1/√(α_t) ≈ 1。

### 2. 数据归一化

**问题**: 动作值在原始弧度空间（范围 [-3, 3], std≈1.6），与 N(0,1) 噪声不匹配。

**修复**: 训练前计算 action mean/std，归一化到零均值单位方差。

```
actions_norm = (actions_raw - mean) / (std + 1e-8)
```

归一化后：
- 数据分布 ~ N(0, 1)（与噪声匹配）
- x_T ~ N(0, I) 的初始分布与目标分布一致
- 推理后反归一化恢复原空间

### 3. Checkpoint 覆盖问题

**问题**: 多次训练共享 `checkpoints/` 目录，best.pt/last.pt 互相覆盖。

**修复**: 每个训练 run 使用独立子目录 `checkpoints/<timestamp>/`，同时保存归一化统计量 (`act_mean`, `act_std`)。

## 踩坑记录

| # | 问题 | 根因 | 解决 |
|---|------|------|------|
| 1 | DDIM 推理输出 ±800 rad | Cosine ᾱ_T≈0 导致除以零 | 改用 DDPM 采样 (100步，数值稳定) |
| 2 | 预测范围/偏移与 GT 不匹配 | 动作值未经归一化 | 训练前计算 mean/std 归一化，推理后反归一化 |
| 3 | GPU 训练被 OOM-kill | `num_workers=4` 多进程 CUDA 上下文耗内存 | CPU 训练 + `num_workers=0` |
| 4 | best.pt 被新训练覆盖 | 多次训练共享同一目录 | 每次训练使用 `checkpoints/<timestamp>/` 子目录 |
| 5 | Python 输出缓冲导致日志为空 | stdout 重定向到文件时缓冲 | 已记录，待修复（加 `-u` flag） |
